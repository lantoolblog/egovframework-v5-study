실습 환경 구성
--------------------------------------------------------------------------------

 1) lab-langchain4j 프로젝트 import
    - IDE에서 Maven 프로젝트로 import
    - Maven Dependencies 다운로드 완료 확인

 2) Groq API 키 발급

    - 상단 메뉴 [API Keys] → [Create API Key] 클릭하여 API 키 발급
    - 무료 티어에서 대부분의 기능(Tool Calling 포함)을 사용할 수 있습니다.

 3) application.yml 설정
    - src/main/resources/application.yml 파일 열기
    - api-key: 항목에 발급받은 API 키 입력
      예) api-key: gsk_xxxxxxxxxxxxxxxxxxxx

 4) 테스트 실행 방법
    - 각 Step의 테스트 클래스(Step1ChatModelTest 등)를 JUnit으로 실행
    - 테스트가 실패하면 해당 Step의 TODO 코드를 먼저 완성하세요.
    - 완성본 확인: lab-langchain4j-tutor 프로젝트의 동일 패키지 참고

 ※ 사용량 제한(Rate Limit) 안내
    Groq 무료 티어는 분당 요청 횟수와 일일 토큰 사용량에 제한이 있습니다.
    테스트 클래스 전체를 한 번에 실행하면 연속 요청으로 인해 아래와 같은
    429 오류가 발생할 수 있습니다.

      HTTP 429 - rate_limit_exceeded

    이 경우 잠시 기다린 후 테스트를 개별 실행하세요.
    각 @Test 메서드를 하나씩 실행하는 것을 권장합니다.

Step 1. ChatLanguageModel 기본 사용
================================================================================

 학습 목표:
  - LangChain4j의 핵심 인터페이스 ChatLanguageModel 이해
  - chatModel.chat(String) : 가장 간단한 AI 호출 (문자열 → 문자열)
  - chatModel.generate(UserMessage) : UserMessage 기반 호출과 Response<AiMessage> 구조 이해
  - 토큰 사용량 메타데이터 확인 (TokenUsage)

 수정 파일: src/main/java/egovframework/lab/ai/step1/service/impl/ChatModelServiceImpl.java
 테스트 파일: src/test/java/egovframework/lab/ai/step1/Step1ChatModelTest.java

 ※ Spring AI와의 차이점:
    Spring AI : chatModel.call(String)  → String 반환
    LangChain4j: chatModel.chat(String) → String 반환

    Spring AI : chatModel.call(Prompt)      → ChatResponse
    LangChain4j: chatModel.chat(UserMessage) → ChatResponse

 -----------------------------------------------------------------------
 [Step 1-1] simpleChat() 구현
 -----------------------------------------------------------------------
  chatModel.chat(String)은 문자열을 입력받아 문자열을 반환하는 가장 간단한 방법입니다.

  /****/
  return chatModel.chat(message);
  /****/

 -----------------------------------------------------------------------
 [Step 1-2] chatWithUserMessage() 구현
 -----------------------------------------------------------------------
  UserMessage를 명시적으로 생성하여 chat()을 호출합니다.
  ChatResponse에는 응답 텍스트 외에 토큰 사용량, 종료 이유 등이 포함됩니다.

  /****/
  return chatModel.chat(UserMessage.from(message));
  /****/

 -----------------------------------------------------------------------
 [Step 1-3] getTotalTokens() 구현
 -----------------------------------------------------------------------
  /****/
  ChatResponse response = chatWithUserMessage(message);
  TokenUsage usage = response.tokenUsage();
  return usage != null ? usage.totalTokenCount() : null;
  /****/

  ※ 모델에 따라 토큰 사용량을 반환하지 않는 경우 null이 반환될 수 있습니다.
     테스트에서 null 처리가 포함되어 있으므로 그대로 실행하면 됩니다.

 → Step1ChatModelTest의 3개 테스트를 개별 실행하여 확인하세요.

Step 2. AiServices
================================================================================

 학습 목표:
  - AiServices 패턴: 인터페이스 선언만으로 AI 연동 구현체 자동 생성
  - @SystemMessage로 시스템 프롬프트 어노테이션 기반 정의
  - @UserMessage로 사용자 메시지 템플릿 정의
  - @V로 템플릿 변수 바인딩
  - AiServices.builder().chatModel().build() 빌더 패턴

 수정 파일: src/main/java/egovframework/lab/ai/step2/service/impl/AiServicesServiceImpl.java
 테스트 파일: src/test/java/egovframework/lab/ai/step2/Step2AiServicesTest.java

 ※ Spring AI와의 차이점:
    Spring AI : ChatClient.Builder를 주입받아 Fluent API 체이닝
    LangChain4j: 내부 인터페이스를 정의하고 AiServices로 구현체를 자동 생성

 -----------------------------------------------------------------------
 [Step 2-1] 내부 인터페이스 정의
 -----------------------------------------------------------------------
  클래스 내부에 private interface를 정의합니다.
  LangChain4j가 런타임에 이 인터페이스의 구현체를 자동으로 생성합니다.

  /****/
  private interface BasicChatbot {
      String chat(String userMessage);
  }

  private interface DynamicSystemChatbot {
      @SystemMessage("{{systemPrompt}}")
      String chat(@V("systemPrompt") String systemPrompt, @UserMessage String userMessage);
  }

  private interface TopicChatbot {
      @UserMessage("{{topic}}에 대해 세 줄로 설명해주세요.")
      String chat(@V("topic") String topic);
  }
  /****/

 -----------------------------------------------------------------------
 [Step 2-2] chat() 구현
 -----------------------------------------------------------------------
  /****/
  BasicChatbot bot = AiServices.builder(BasicChatbot.class)
          .chatModel(chatModel)
          .build();
  return bot.chat(message);
  /****/

 -----------------------------------------------------------------------
 [Step 2-3] chatWithSystemPrompt() 구현
 -----------------------------------------------------------------------
  /****/
  DynamicSystemChatbot bot = AiServices.builder(DynamicSystemChatbot.class)
          .chatModel(chatModel)
          .build();
  return bot.chat(systemPrompt, userMessage);
  /****/

 -----------------------------------------------------------------------
 [Step 2-4] chatAboutTopic() 구현
 -----------------------------------------------------------------------
  /****/
  TopicChatbot bot = AiServices.builder(TopicChatbot.class)
          .chatModel(chatModel)
          .build();
  return bot.chat(topic);
  /****/

Step 3. PromptTemplate
================================================================================

 학습 목표:
  - PromptTemplate으로 재사용 가능한 프롬프트 구조화
  - {{variable}} 이중 중괄호 변수 문법 (Spring AI의 {variable}과 다름!)
  - Map.of()를 사용한 단일/복수 변수 바인딩
  - SystemMessage와 UserMessage를 조합한 Prompt 구성

 수정 파일: src/main/java/egovframework/lab/ai/step3/service/impl/PromptTemplateServiceImpl.java
 테스트 파일: src/test/java/egovframework/lab/ai/step3/Step3PromptTemplateTest.java

 ※ Spring AI와의 가장 큰 차이점:
    Spring AI : "{variable}"  (단일 중괄호)
    LangChain4j: "{{variable}}" (이중 중괄호)

 -----------------------------------------------------------------------
 [Step 3-1] askWithSingleVariable() 구현
 -----------------------------------------------------------------------
  /****/
  PromptTemplate template = PromptTemplate.from("{{adjective}} 프로그래밍 언어 하나를 추천하고 이유를 설명해주세요.");
  Prompt prompt = template.apply(Map.of("adjective", adjective));
  return chatModel.chat(prompt.toUserMessage()).aiMessage().text();
  /****/

 -----------------------------------------------------------------------
 [Step 3-2] askWithMultipleVariables() 구현
 -----------------------------------------------------------------------
  /****/
  PromptTemplate template = PromptTemplate.from("{{country}}의 {{category}} TOP 3를 알려주세요.");
  Prompt prompt = template.apply(Map.of("country", country, "category", category));
  return chatModel.chat(prompt.toUserMessage()).aiMessage().text();
  /****/

 -----------------------------------------------------------------------
 [Step 3-3] askWithSystemAndUser() 구현
 -----------------------------------------------------------------------
  /****/
  PromptTemplate systemTemplate = PromptTemplate.from("당신은 {{field}} 분야의 전문가입니다. 모든 답변은 한국어로 해주세요.");
  String systemText = systemTemplate.apply(Map.of("field", field)).text();
  SystemMessage systemMessage = SystemMessage.from(systemText);

  PromptTemplate userTemplate = PromptTemplate.from("{{topic}}에 대해 핵심만 세 문장으로 설명해주세요.");
  UserMessage userMessage = userTemplate.apply(Map.of("topic", topic)).toUserMessage();

  return chatModel.chat(List.of(systemMessage, userMessage)).aiMessage().text();
  /****/

Step 4. Structured Output
================================================================================

 학습 목표:
  - AI 응답을 Java 객체(record)로 직접 받는 방법 이해
  - AiServices의 반환 타입을 POJO로 선언하면 자동으로 JSON 파싱
  - List<T> 직접 반환 불가 이유 이해 (JSON Schema 표준 제약)
  - 래퍼 record로 List를 감싸서 처리하는 패턴 학습
  - JSON 파싱 코드 없이 타입 안전한 AI 응답 처리

 수정 파일: src/main/java/egovframework/lab/ai/step4/service/impl/StructuredOutputServiceImpl.java
 테스트 파일: src/test/java/egovframework/lab/ai/step4/Step4StructuredOutputTest.java

 참고: BookInfo.java (record)는 이미 작성되어 있습니다.
       AI 응답이 이 record의 필드 구조에 맞게 역직렬화됩니다.

 ※ Spring AI와의 차이점:
    Spring AI : .entity(BookInfo.class) 또는 .entity(new ParameterizedTypeReference<List<BookInfo>>() {})
    LangChain4j: AiServices 인터페이스의 반환 타입으로 BookInfo 선언
                 단, List<BookInfo> 직접 반환은 불가 → 래퍼 record 필요 (아래 참고)

 -----------------------------------------------------------------------
 [Step 4-1] 래퍼 record와 내부 인터페이스 정의
 -----------------------------------------------------------------------
  List<BookInfo>를 AiServices 인터페이스의 반환 타입으로 직접 사용하면
  PojoCollectionOutputParser.formatInstructions()에서 IllegalStateException 발생.
  이는 JSON Schema 표준이 최상위 배열([...])을 지원하지 않기 때문.
  List를 필드로 감싸는 래퍼 record를 선언하면 {"books": [...]} 형태로 처리 가능.

  /****/
  private record BookInfoList(List<BookInfo> books) {}

  private interface BookInfoExtractor {
      BookInfo extractBookInfo(@UserMessage String prompt);
      BookInfoList extractBookList(@UserMessage String prompt);
  }
  /****/

 -----------------------------------------------------------------------
 [Step 4-2] getBookInfo() 구현 - 단일 객체
 -----------------------------------------------------------------------
  /****/
  BookInfoExtractor extractor = AiServices.builder(BookInfoExtractor.class)
          .chatModel(chatModel)
          .build();
  return extractor.extractBookInfo("'" + bookTitle + "' 책에 대한 정보를 알려주세요.");
  /****/

 -----------------------------------------------------------------------
 [Step 4-3] getBooksByAuthor() 구현 - List 타입
 -----------------------------------------------------------------------
  /****/
  BookInfoExtractor extractor = AiServices.builder(BookInfoExtractor.class)
          .chatModel(chatModel)
          .build();
  return extractor.extractBookList("작가 '" + authorName + "'의 대표작 3권을 목록으로 알려주세요.").books();
  /****/

Step 5. Tool Calling
================================================================================

 학습 목표:
  - @Tool 어노테이션으로 Java 메서드를 AI 도구로 등록
  - @P로 파라미터 설명 제공
  - AiServices에 .tools()로 Tool 등록
  - AI가 Tool을 자율적으로 선택하고 호출하는 흐름 이해

 ※ 모델 확인: application.yml의 model을 llama-3.3-70b-versatile 로 변경하세요.
    llama-3.1-8b-instant는 Tool Calling을 안정적으로 지원하지 않을 수 있습니다.
    application.yml에서 주석 처리된 줄을 참고하세요.

 ※ Spring AI와의 차이점:
    Spring AI : @Tool(description = "설명")  @ToolParam(description = "설명")
    LangChain4j: @Tool("설명")               @P("설명")
